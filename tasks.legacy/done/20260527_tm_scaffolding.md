# Project scaffolding and pytest harness

## Problem statement

The feature requires a Python package installable via `uv tool install -e ./taskman`, with `typer` CLI, `ruamel.yaml` for YAML I/O, and `pytest` as the test framework. Without scaffolding, no command can be implemented and no test can run.

## Scope

**In:** Top-level `taskman/` directory at the repo root (sibling of `plugins/`); `pyproject.toml` declaring runtime deps (`typer`, `ruamel.yaml`) and dev deps (`pytest`); src layout `taskman/src/taskman/{__init__.py, cli.py}` with placeholder `model/` and `commands/` package directories; typer entry point registered as `taskman` console script; pytest harness at `taskman/tests/` with a smoke test that imports the package and invokes `taskman --help`; verification that `uv tool install -e ./taskman` succeeds and exposes `taskman` on PATH.

**Out:** any business logic (model primitives, real commands); bash shim; documentation; CI configuration.

## Acceptance criteria

1. `uv tool install -e ./taskman` succeeds and `taskman` is on PATH.
2. `taskman --help` renders typer's rich help with a placeholder Hello-world-style command or no commands (typer-default help).
3. `pytest taskman/tests/` runs and the smoke test passes.
4. Package layout matches `taskman/src/taskman/{__init__.py, cli.py, model/__init__.py, commands/__init__.py}` and `taskman/tests/__init__.py`.
5. The existing `plugins/agn/scripts/taskman.sh` is untouched.

## Quality gates

- `uv tool install -e ./taskman` exits 0 with no warnings.
- `taskman --help` exits 0.
- `pytest` exits 0.
- No imports of project-internal code outside `taskman/`.

### Notable decisions
- **`@app.callback()` shim** — typer collapses a single `@app.command()` into the root command. Adding an empty `@app.callback()` forces multi-command dispatch so `taskman version` is a real subcommand (caught during smoke-test failure on first install; fixed in same session).
- **src layout** chosen over flat `taskman/taskman/` for clean test isolation and to keep `tests/` importing only via installed package.
- **hatchling** as build backend — uv-recommended default; minimal config for src layout (`packages = ["src/taskman"]`).
- **typer testing via `CliRunner`** — preferred over subprocess invocation for unit-test speed and isolation.
