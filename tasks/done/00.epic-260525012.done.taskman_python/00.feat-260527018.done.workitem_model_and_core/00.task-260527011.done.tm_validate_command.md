# validate command for model integrity

## Problem statement

The model splits identity across the on-disk name and the YAML frontmatter for cross-check. The two surfaces can drift if anything other than `taskman` writes them. `validate` is the integrity gate that detects drift, broken grammar, orphan paths, and other model violations before they propagate.

## Scope

**In:** `taskman validate [--json]` subcommand; walk the entire nested tree; per-item checks (name parses; frontmatter `id`/`slug`/`status`/`priority`/`type` agrees with name tokens; parent directory's implied parent ID matches an existing item or top-level status folder; `task = leaf` invariant holds — no task item is a directory and no task item has children); structured error/warning output; pytest covering each violation type.

**Out:** dependency-graph checks (separate feature `schema_dependencies`); fix-up logic (`validate` is read-only); migration verification (separate feature).

## Acceptance criteria

1. `taskman validate` exits 0 on a healthy tree and prints a clean summary.
2. Exits non-zero on any error; prints per-item findings with file path and the specific drift detected.
3. Detects name↔frontmatter drift in `status`, `slug`, `priority`, `id`, `type`.
4. Detects broken name grammar (missing field, wrong charset, malformed type/status/priority token).
5. Detects orphan parent reference (file under a nonexistent directory ID or under no recognized parent at all).
6. Detects `task = leaf` violation (a task item that is a directory, or a task item with children).
7. `--json` returns structured findings: list of `{path, severity, code, message}`.
8. pytest covers each violation type plus the all-clean case.

## Quality gates

- pytest passes.
- `validate` performs zero writes.
- Severity levels distinct: error blocks exit 0; warning does not.

### Notable decisions
- **11 violation codes** rather than a single generic "drift" code — gives users a searchable handle and lets the JSON-mode consumer route by code.
- **Sorted tree walk** (`sorted(tasks_dir.rglob("*"), key=lambda p: str(p))`) for deterministic output ordering — important for human review and for future test stability.
- **Warning severity for `missing_field`** (frontmatter omits a name field) — does not block exit 0. Real drift is name↔YAML mismatch, which is error. A missing field is recoverable; closing the gap on read is also viable.
- **`validate` does zero writes.** Pure-read traversal; the only output is to stdout/stderr.
