# move command for top-level status transitions

## Problem statement

A top-level work item changes status by being moved between `backlog/`, `active/`, `done/` folders. The status token in its name must update atomically with the folder move; otherwise the in-name status token and the folder diverge and `validate` will report drift.

## Scope

**In:** `taskman move <id> <backlog|active|done>` subcommand; atomic folder move + in-name status token rewrite; preservation of any children (subtree moves with the root; children retain their own status tokens in their names); rejection when target equals current status; pytest covering each path.

**Out:** lifecycle preconditions for done (handled by `close`); per-child status updates; non-top-level moves (children change status via `convert`-style operations not in this task's scope, deferred).

## Acceptance criteria

1. `taskman move <id> active` moves a backlog top-level item to `active/`, rewriting the in-name status token from `backlog` to `active`.
2. Subtree moves with the root: child files and folders are not renamed; only the top-level item's directory/file is repositioned and its name updated.
3. `taskman move <id> <current-status>` is a no-op with a clear message and exit 0.
4. Moving a non-existent ID returns a clear error and exits non-zero.
5. pytest covers backlog→active, active→done, no-op, missing ID, and subtree-preservation cases.

### Notable decisions
- **Rejects nested items.** Only top-level items move via `move`; nested items keep their status in their name and change it via `convert`-style rewrites (or by moving with the root).
- **Atomic rename.** Single `Path.rename` (POSIX `rename(2)`) moves the whole subtree atomically when the item is a directory; for files it's a one-syscall move. The YAML status update afterward is also atomic per-file. The two operations are sequenced but each is atomic — interruption between them leaves a discoverable drift (name token mismatches YAML status) that `validate` will catch.
- **Children retain their own status tokens.** Verified by `test_move_preserves_subtree` — child's in-name status remains `backlog` even when the root moves to `active`.
