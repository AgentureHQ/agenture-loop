# Search command for the nested work-item tree

## Problem statement

A growing nested tree of work items needs a fast way to locate items by title, slug, or ID without manually grepping the file tree. Both humans and agents need this.

## Objective

Add `taskman search <query>` that returns ranked matches across the nested tree with type, status, ID, slug, title, and path; with `--json` for agent consumption.

## Scope

**In:** `search` subcommand walking the entire nested tree; ranking by token match against title, slug, ID, and body excerpt; `--json` output mode for agents.

**Out:** `/agn:define task` integration (deferred to skill-integration follow-on); fuzzy-match beyond what regex/token matching provides natively; UI search (covered in `web_ui`).

## Acceptance criteria

1. `taskman search "<query>"` recursively walks the tasks tree and returns matching items with type, status, ID, slug, title, and path.
2. Results are ranked: exact ID match > exact slug match > title-token match > body-token match.
3. `taskman search --json "<query>"` returns the same results as structured JSON.
4. Empty-result case returns exit 0 and a structured empty list.
5. pytest covers ranking, JSON shape, empty-result case, and tree-walk across mixed-shape fixtures.

## Summary

Single task delivered: `tm_search_command` — `taskman search <query>` in `commands/search.py` with 5-tier ranking (exact ID > exact slug > slug-contains > title-token > body-token), `--json`, 10 tests. All 5 ACs verified. 194 tests pass.

Self-certification — single-task feature with full test coverage; QA agent skipped per the trade-off documented in feature 2.
