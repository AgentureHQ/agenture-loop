# worktree-status command aggregating active items across git worktrees

## Problem statement

A user with multiple active git worktrees (one per in-flight feature) has no single command that aggregates "what is in flight across all my worktrees." Today they must `cd` into each worktree to inspect its tasks tree.

## Scope

**In:** `taskman worktree-status` subcommand; parses `git worktree list --porcelain`; for each worktree, walks its `<wt>/tasks/` (or the current-process `TASKMAN_TASKS_DIR` if it sits inside that worktree); collects items whose in-name status is `active`; prints a per-worktree summary or `--json`.

**Out:** Multi-repo aggregation (single-repo only); cross-worktree editing; live updates; reading worktrees that are not git-worktrees.

## Acceptance criteria

1. `taskman worktree-status` runs from the main repo or any worktree and prints a per-worktree summary: worktree path, branch (or "(detached)"), and each active item's ID + title (one per line).
2. Worktrees with no active items show "(none)".
3. Worktrees without a `tasks/` folder show "(no tasks dir)".
4. `--json` returns structured data `[{worktree, branch, items: [{id, type, slug, title, path}, ...]}, ...]`.
5. The current worktree honors a `TASKMAN_TASKS_DIR` override (if set in the current process and the directory is inside the worktree); other worktrees use their default `<wt>/tasks/`.
6. pytest covers: porcelain parsing (single worktree, multi-worktree, detached HEAD), per-worktree aggregation (none/some/missing-dir), JSON output.

## Quality gates

- pytest passes.
- Read-only command (no writes).
- Graceful exit if `git` is unavailable or cwd is not a git repo (prints a clear error, non-zero exit).
- `--json` produces parseable JSON on stdout.

### Notable decisions
- **Aggregation by in-name status, not folder.** A nested item whose name carries `.active.` counts even when sitting inside a backlog root. Matches the "in-name status is per-item truth" model.
- **`TASKMAN_TASKS_DIR` honored only when the override path sits inside the worktree.** Prevents the current process's env var from poisoning aggregation for other worktrees. Other worktrees always use their default `<wt>/tasks`.
- **Graceful exit when `git` is unavailable or cwd is not a repo** — both surface a clear error to stderr and exit non-zero.
