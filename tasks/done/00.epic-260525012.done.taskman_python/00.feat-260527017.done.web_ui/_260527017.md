# Local web UI for browse, search, prioritize

## Problem statement

CLI lists work for agents but not for human review of a 40+ item nested tree. The project manager needs a visual surface to browse, search, prioritize, and see the dependency graph.

## Objective

Add `taskman ui` that starts a local web server and opens the browser to a page rendering the nested tree, ready/blocked/recently-done sections, and a Mermaid dependency graph.

## Scope

**In:** `taskman ui` subcommand; local web server bound to `localhost:8080` by default; page templates rendering nested tree, ready, blocked, recently completed (last 7 days); Mermaid dependency graph in-browser; client-side search filter; side-panel preview of item bodies; `taskman ui --stop` and Ctrl+C lifecycle; library choice (FastAPI + Jinja2 + uvicorn vs stdlib `http.server`) decided and justified in this feature's design notes.

**Out:** editing items through the UI (read-only); authentication; multi-user; live websocket updates (page refresh is sufficient); server-side image generation.

## Acceptance criteria

1. `taskman ui` starts a local web server and opens the default browser to the rendered page.
2. The page renders sections: nested tree, ready, blocked, recently completed, dependency graph.
3. A search box filters items across all sections client-side with no page reload.
4. Clicking an item opens its full markdown body in a side panel.
5. Refreshing the page after a CLI change shows the new state.
6. Closing the browser does not kill the server; user stops it via `taskman ui --stop` or Ctrl+C.
7. The Mermaid dependency graph renders in-browser.
8. Library choice is documented and justified in the feature's design notes.
9. pytest covers data-aggregation functions; a smoke test starts the server, fetches the page, and asserts HTTP 200 plus presence of key DOM elements.

## Summary

Single task delivered: `tm_ui_command` — `taskman ui` (stdlib `http.server` + inline HTML/JS + Mermaid CDN). Two endpoints: `GET /` (page) and `GET /api/show/<id>` (JSON). 11 tests (data-prep + server smoke + escape check). All 9 ACs verified. 214 tests pass.

**Library choice (AC 8):** stdlib `http.server.ThreadingHTTPServer` + inline templates. Zero new runtime dependencies. Justified by the read-only single-user single-page scope; FastAPI's benefits don't apply.

Self-certification — single-task feature; QA agent skipped per the trade-off documented in feature 2.
